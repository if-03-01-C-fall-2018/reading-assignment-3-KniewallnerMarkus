The Unified Resource Locator

General Indroduction

A Uniform Resource Locator (URL), colloquially termed a web address,[1] is a reference to a web resource that specifies its
location on a computer network and a mechanism for retrieving it.

A URL is a specific type of Uniform Resource Identifier (URI),[2][3] although many people use the two terms interchangeably.

[4][a] URLs occur most commonly to reference web pages (http), but are also used for file transfer (ftp), email (mailto),
database access (JDBC), and many other applications.

Most web browsers display the URL of a web page above the page in an address bar. A typical URL could have the form http://www.example.com/index.html, which indicates a protocol (http), a hostname (www.example.com), and a file name (index.html).


Section about the Syntax

Every HTTP URL conforms to the syntax of a generic URI. The URI generic syntax consists of a hierarchical sequence of five components:

URI = scheme:[//authority]path[?query][#fragment]


Where the authority component divides into three subcomponents:

authority = [userinfo@]host[:port]


Web Servers

1. The primary function of a web server is to store, process and deliver web pages to clients.

2. While the primary function is to serve content, a full implementation of HTTP also includes ways of receiving content from clients.
This feature is used for submitting web forms, including uploading of files.

3. Many generic web servers also support server-side scripting using Active Server Pages (ASP), PHP, or other scripting languages.


Web Client (Web Browser)


General Indroduction

A web browser (commonly referred to as a browser) is a software application for accessing information on the World Wide Web. Each individual web page, image, and video is identified by a distinct URL, enabling browsers to retrieve and display them on the user's device.

Common user interface features of browsers:

All major browsers allow the user to open multiple pages at the same time, either in different browser windows or in different tabs of the same window. They also support the use of extensions to add to or modify browser operation in a variety of ways.

1. Back and forward buttons to go back to the previous page visited or forward to the next one.

2. A refresh or reload button to reload the current page.

3. A stop button to cancel loading the page. (In some browsers, the stop button is merged with the reload button.)

4. A home button to return to the user's home page.

5. An address bar to input the URL of a page and display it.

6. A search bar to input terms into a search engine. (In some browsers, the search bar is merged with the address bar
